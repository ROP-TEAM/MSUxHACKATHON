# Product

## Register

product

## Users

สองกลุ่มหลัก:
- **ผู้ใช้ทั่วไป**: คนดูคอนเสิร์ตที่ต้องการค้นหา Event, ดูรายละเอียด, ซื้อบัตร
- **แอดมิน**: ผู้จัดงานที่ต้องดูภาพรวมยอดขาย, สถิติ, จัดการข้อมูล

Context: ใช้งานในมือถือ/ desktop, เน้นค้นหาและดูข้อมูลเร็ว

## Product Purpose

MSU x Hackathon — แพลตฟอร์มบริหาร Event คอนเสิร์ตและบัตรเข้าชม เอาข้อมูลดิบ (JSON) มาแสดงผลเป็น Dashboard ที่ใช้งานได้จริง พร้อม AI ช่วยวิเคราะห์และตอบคำถาม

Success criteria: ข้อมูลดูง่าย, ค้นหาได้, กรองได้, มีกราฟ KPI, AI ช่วยวิเคราะห์

## Brand Personality

- **Playful** — สนุก ไม่เคร่งขรึม
- **Energetic** — มีชีวิตชีวา สีสันสดใส
- **Youthful** — ดูทันสมัย วัยรุ่น

Emotion target: ตื่นเต้นเวลาดูคอนเสิร์ต, รู้สึกว่าข้อมูลเข้าถึงง่าย สนุกกับการใช้งาน

## Anti-references

- ไม่เอา SaaS corporate template (เยอะเกิน, ดูแข็ง)
- ไม่เอา ดีไซน์มืดหม่น
- ไม่เอา animation เยอะจนรก

## Design Principles

1. **Playful precision** — สนุกแต่ไม่เลอะเทอะ ทุก element มีเหตุผล
2. **Data-first clarity** — ข้อมูลต้องอ่านง่าย เข้าใจได้ใน 3 วิ
3. **AI as companion** — AI ช่วยเสริม ไม่ใช่บังคับให้ใช้
4. **Mobile-conscious** — hackathon แต่ responsive จำเป็น
5. **Motion with purpose** — animation ช่วยเล่าเรื่อง ไม่ใช่แค่ตกแต่ง

## Accessibility & Inclusion

- WCAG AA (4.5:1 body text, keyboard navigation, focus indicators)
- รองรับ reduced motion
- สีไม่พึ่งพา hue อย่างเดียว (ใช้ icon + text ประกอบ)
